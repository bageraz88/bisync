# Contributing to Bisync Control Center

Thank you for your interest in contributing! 🎉  
We welcome improvements, bug fixes, and new ideas to make Bisync even better.

---

## How to Contribute

1. **Fork the repository**  
   Create your own copy of the project under your GitHub account.

2. **Create a branch**  
   Use a descriptive branch name:
   - `feature/gui-enhancement`
   - `fix/log-rotation-bug`
   - `docs/update-readme`

3. **Make your changes**  
   - Keep commits focused and atomic.  
   - Follow existing code style and conventions.  
   - Update documentation if your changes affect usage.

4. **Test thoroughly**  
   Ensure your changes don’t break existing functionality:
   - Run scripts locally (`setup-bisync.sh`, `build-appimage.sh`).  
   - Verify both CLI and GUI modes.  
   - Check log rotation and archive scripts.

5. **Commit messages**  
   Use clear, conventional commit messages:
   - `feat: add conflict popup to GUI`
   - `fix: correct log archive date format`
   - `docs: update Quick Start section`

6. **Push and open a Pull Request (PR)**  
   - Provide a clear description of your changes.  
   - Reference related issues if applicable.  
   - Include screenshots or logs for GUI/CLI changes.

---

## Contribution Areas

- **Scripts** → backend automation (`bisync-status.sh`, `bisync-report.sh`, etc.)  
- **GUI** → QML frontend (`main.qml`)  
- **Build** → AppRun logic, setup/build scripts  
- **Docs** → README, LICENSE, CONTRIBUTING  

---

## Code of Conduct

- Be respectful and collaborative.  
- Provide constructive feedback.  
- Keep discussions focused on improving the project.
